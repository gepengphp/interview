# LDAP

## AD

**全称**：Active Dritectory（活动目录）。   
常与 LDAP 同时出现，是 LDAP 的一个常见应用。   
`AD` 实现了 `LDAP` 协议服务器，在 LDAP 基础上实现了 `域控`（Domain Controller）的功能。是windows Server 2000操作系统平台的中心组件之一。   
**个人理解**：对外提供 LDAP 协议的身份认证，对资源进行统一管理，可以与第三方应用集成。   
**个人举例**：如一家公司的域、组织架构、用户、用户信息、账号密码、打印机、计算机等，都可以存储在 LDAP 服务中，供这个公司的第三方应用查询，作为认证数据源。   


# LDAP

### 简介
**读作**：/ˈɛldæp/。   
**全称**：Lightweight Directory Access Protocol（轻量级目录访问协议）  
LDAP 是一个目录服务协议，主要用于与查询，如密码验证、通讯录、组织架构…。   
本质可以理解为一个 `数据库`。

### 基本模型
- `条目`（entry）：一个或多个`属性`的集合，用一个全局唯一的 `dn` 标识。   
    `条目`相当与数据库中的一条记录。
- `dn`：全局唯一的 `dn` 标识，`dn` 相当于记录主键。
- `ObjectClass`：对象类。
