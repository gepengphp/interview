# 灵异事件

## points
> 背景：数据库某字段保存一个 json 格式的字符串，其结构为：
> ```json
> {
>    "points":[
>       {"x":0,"y":1},
>       {"x":2,"y":3},
>       {"x":4,"y":5},
>       {"x":6,"y":7}
>     ]
> }
> ```
> 前端框架是：`vue` + `iview` + `axios`。  

灵异：妈蛋的我返回的是上面的结构，但从 `axios` 的 `response` 拦截器中打印出来的结果变成了这个样子：
```
points: Array(4)
0: {x: -57, y: -56}
1: {x: -17, y: -16}
2: {x: 23, y: 24}
3: {x: 63, y: 64}
length: 4
```

这个结果看起来有规律，比如上面的 `x` 是一个等差数列。但是实际上的规律更复杂。因为结果会根据不同的初始数值转换成不同的数字。
比如把 `points[0]['x']` 赋值为 `1` 后，打印的结果是：
```
points: Array(4)
0: {x: -37, y: -56}
1: {x: -17, y: -16}
2: {x: 23, y: 24}
3: {x: 63, y: 64}
length: 4
```

而且改变 `x`、`y` 的键名没有用，改变 `x`、`y` 的值，如果是 `Number` 没问题，如果是字符串，比如把 `points[0]['x']` 赋值为 `1a`，哪怕只改了其中一个，也会影响所有的 `x`、`y` 的值变成 `NaN`。不知道中间经历了什么样的过程。
